# Peer Evaluation Kit

Use this to ask real users to install, run, and post practical notes.

## Short DM

```text
我做了一个 OPC 参赛 Skill，叫「一人公司售前作战室」。

它不是写销售鸡汤，而是把潜在客户线索和客户聊天记录转成：破冰话术、跟进节奏、报价边界、成交台账和可复用经验。

如果你有获客、接单、报价、客户拖延、免费 Demo、分成合作这类经历，可以拿一段匿名线索或聊天试一下。跑完后如果有用，帮我发一条 BotLearn 实战笔记：写你输入了什么、它帮你解决了什么。
```

## Longer DM

```text
我现在在参加 BotLearn 的 OPC Agent Skills 比赛，做了一个 Skill：一人公司售前作战室。

它面向 AI 自由职业者、一人公司和小型工作室，解决的不是泛泛销售问题，而是这些具体场景：
- 不知道第一封获客消息怎么写
- 客户问价格时不知道怎么拆报价
- 客户想免费完整 Demo
- 客户拖资料/拖付款
- 客户提出后续分成替代当前费用
- 每次谈单经验都没有沉淀

你可以拿一段匿名客户聊天或潜在客户线索试一下。跑完后，如果它给出的下一步动作、话术或经验沉淀有帮助，麻烦在 BotLearn 社区发一条实战笔记。

实战笔记不用很长，只要写：
1. 你输入了什么场景
2. Skill 输出了什么
3. 哪个建议你觉得有用
4. 你准备怎么应用
```

## Example If They Have No Case Ready

```text
潜在客户类型：
小型品牌方，有官网/社媒内容，但页面展示和客户跟进路径比较弱。

我能提供：
AI 工作流、品牌独立站、内容自动化和售前跟进经验沉淀。

我最纠结的问题：
我不想一上来就说“我能帮你做 AI”，想找到一个客户更容易理解的切入点。
```

## Follow-Up Message After They Try It

```text
你刚才试出来的结果里，最有用的是哪一块？

如果方便的话，你可以直接按这个结构发一条实战笔记：
1. 你输入的匿名场景
2. 它给的下一步动作/话术
3. 你觉得能帮你避免什么坑

不用写很长，真实使用感比长文更重要。
```

## Who To Ask First

Prioritize people with real current cases:

| Priority | Who |
|---|---|
| 1 | AI freelancer with active client chats |
| 2 | solo founder doing outbound or service sales |
| 3 | designer/developer taking website or automation projects |
| 4 | content/marketing operator dealing with leads |
| 5 | other OPC contest participants willing to互装互评 |

Avoid asking people who have no client-facing context; their notes will be weaker.
