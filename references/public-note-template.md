# Public Note Template

Use this reference when preparing public usage notes, demo scripts, screenshots, project page copy, or shareable case write-ups.

## Privacy

Before writing public notes:

- anonymize client names
- remove exact private chat identifiers
- round or generalize sensitive prices if needed
- avoid exposing client internal plans
- show the workflow, not private gossip

## Public Note Structure

```markdown
# 我用「成交信号捕手」处理了一次真实客户跟进

## 背景
我是一个 <AI 自由职业者/一人公司/工作室>，最近遇到的问题是：<一句话痛点>。

## 输入
我给 Skill 的材料包括：
- 客户项目类型：
- 客户最新表达：
- 我已经交付/沟通到哪一步：
- 我最纠结的问题：

## Skill 输出了什么
它没有只写一段话术，而是给了：
1. 客户阶段判断
2. 风险点
3. 下一步动作
4. 可直接发送话术
5. 报价/边界建议
6. 可沉淀的成交台账

## 我实际采用的动作
<写你选择了哪条话术、改了哪里、发没发>

## 对一人公司的价值
这次最有用的是：<减少犹豫/避免免费 Demo/明确报价边界/沉淀经验>。
```

## 60-90 Second Demo Script

```text
我做这个 Skill 是因为一人公司最大的问题不是不会干活，而是客户来了以后不会判断意向、报价和设边界。

现在我输入一段客户消息：客户希望费用优化，并提出后续按网页营业额分成。

Skill 会先给结论：这不是直接拒绝信号，而是客户想降低前期现金支出，用未来不确定收益替代当前建设费。

接着它给下一步动作：不要继续证明能力，要把基础建设费和后续分成拆开。

然后它给一段可以直接发给客户的微信话术，再生成报价边界、成交台账和一段公开复盘草稿。

所以它不是普通销售话术生成器，而是一个一人公司的售前判断框架：帮你把客户跟进变成可复用的经验资产。
```
