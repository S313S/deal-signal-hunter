# Acquisition Playbook

Use this reference when the user needs client acquisition, lead research, cold outreach, LinkedIn/Xiaohongshu/WeChat first contact, or a lightweight follow-up sequence.

## Principle

Acquisition output must be narrow enough to send today.

Do not produce a generic marketing plan. Produce:

1. who to approach
2. why now
3. what pain to mention
4. what first message to send
5. when to follow up
6. what reusable acquisition lesson to keep

## Lead Research Frame

| Field | What to extract |
|---|---|
| 客户类型 | industry, business model, size, public signals |
| 可能痛点 | slow content production, weak site, poor follow-up, high support load |
| 可切入服务 | AI workflow, brand website, content acquisition, support automation |
| 信任证据 | relevant case, demo, before/after artifact, short workflow map |
| 首步动作 | first message, sample audit, 15-minute call, paid test |

## Lead Scoring

Score each lead 1-5:

| Score | Meaning |
|---|---|
| 5 | clear pain, public activity, likely budget, decision maker reachable |
| 4 | clear pain and reachable, budget unknown |
| 3 | possible pain, unclear priority |
| 2 | weak signal, needs nurturing |
| 1 | poor fit or no current trigger |

Recommend approaching only score 4-5 first.

## First Message Formula

Use `specific observation + business pain + small next step`.

```text
你好，我看了你们最近的 <官网/内容/活动/产品页>，感觉现在最值得优化的不是单纯多发内容，而是把 <获客/展示/客服/跟进> 做成更稳定的转化流程。

我这边做的是 AI 工作流和一人公司售前自动化，可以先基于你们公开信息做一个很小的示例：<具体输出>。

如果你愿意，我可以先发你一页简版，不占用太多时间。
```

## Three-Step Follow-Up

| Touch | Timing | Goal | Message angle |
|---|---|---|---|
| 1 | Day 0 | open a relevant pain | one observation + one small offer |
| 2 | Day 2-3 | add proof | show a similar mini case or before/after |
| 3 | Day 6-7 | clean close | ask if worth pausing or scheduling |

Do not chase indefinitely. If no response after three touches, mark as nurture.

## Acquisition Memory

Every outreach run should leave:

```markdown
### 获客经验：<一句话规则>

- 目标客户：
- 触发信号：
- 有效破冰点：
- 无效说法：
- 下次复用场景：
```

Example:

```markdown
### 获客经验：不要卖 AI，卖客户看得懂的流程改善

- 目标客户：有品牌展示或内容更新压力的小型品牌方。
- 触发信号：官网/内容长期不更新，但近期有活动或新品。
- 有效破冰点：指出展示、跟进、内容复用的具体断点。
- 无效说法：直接说我能帮你做 AI 自动化。
- 下次复用场景：客户公开内容多，但转化路径和跟进动作弱。
```

## Case Patterns To Mention

Use these as abstract patterns, not fake credentials:

| Pattern | How to apply |
|---|---|
| PitchSense-style | match target, generate value proposition, produce one-page outreach asset |
| Creator-connector-style | research brand, match collaboration targets, write personalized first message |
| PresentMe-style | turn raw project material into proposal/demo/case page |
| Heydesk-style | turn knowledge and chats into support/follow-up workflow |

Do not claim the user built those projects unless they actually did. Say "this workflow borrows the pattern of..." when needed.
