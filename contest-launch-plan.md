# Contest Launch Plan

Goal: push `solo-sales-war-room` toward top-3 by optimizing for usefulness, install conversion, and practical-note volume.

## Positioning

Use this one-liner everywhere:

```text
一人公司售前作战室：把潜在客户线索和客户聊天记录转成破冰话术、跟进节奏、报价边界、成交台账和可复用经验。
```

Avoid vague positioning:

- "AI 销售助手"
- "客户跟进工具"
- "帮你谈单"

Those are too generic. The stronger claim is: **turn each lead or client chat into a reusable sales asset.**

## Scoring Strategy

Competition mechanics reward install volume and practical notes. Therefore:

| Target | What to optimize |
|---|---|
| installs | clear pain, simple default prompt, one screenshot outcome |
| practical notes | every user can post one real before/after case |
| finalist demo | show a sharp client scenario, not a generic feature tour |

The skill must be presented as acquisition + follow-up, not only closing.

## Demo Scenario

Use the revenue-share case because it is easy to understand:

```text
Customer says: Can the price be optimized? Later you can take a percentage of website revenue.
```

The demo must show three things within 90 seconds:

1. The skill identifies the real risk: future uncertain revenue replacing current build fee.
2. The skill gives a sendable message, not just analysis.
3. The skill stores reusable experience for the next similar client.

If judges ask about acquisition, switch to this mini-demo:

```text
I have a small brand lead with weak website/content follow-up. Generate target pain, first outreach message, three-step follow-up, and reusable acquisition lesson.
```

## Submission Page Checklist

Use `submission.md` as the base. Before submitting, make sure the page has:

- one-sentence value proposition
- target users: AI freelancers / one-person companies / small studios
- one concrete pain: pricing, boundaries, follow-up, deal memory
- one acquisition pain: who to approach and what first message to send
- screenshots of output sections:
  - `结论`
  - `下一步动作`
  - `可直接发送的话术`
  - `成交台账记录`
  - `可复用经验`
- link or instructions for the demo scenario

## Practical Note Campaign

Ask early users to post one of these note formats.

### Note 1: Revenue Share Boundary

```markdown
# 我用一人公司售前作战室处理了一次“后续分成”谈判

客户希望当前费用优化，并提出后续按营业额分成。Skill 帮我判断这不是直接拒绝，而是客户想用未来不确定收益替代当前建设费。

我最终采用的动作是：当前基础建设费先结算，后续分成等上线、有可核算订单后再谈。最有用的是，它还生成了成交台账和可复用经验，下次遇到类似客户可以直接复用。
```

### Note 1B: First Outreach

```markdown
# 我用一人公司售前作战室生成了一条获客破冰消息

我输入了一个小型品牌线索：有官网/社媒内容，但展示和跟进路径比较弱。Skill 没有给泛泛营销建议，而是输出目标客户画像、最值得切入的痛点、首封破冰话术和三步跟进节奏。

对一人公司来说，这比“多发私信”更有用，因为它先判断为什么这个客户值得联系，再告诉我第一句话怎么说。
```

### Note 2: Free Demo Boundary

```markdown
# 我用一人公司售前作战室避免了一次免费完整 Demo

客户想先看完整效果再决定是否合作。Skill 没有建议我继续证明能力，而是把下一步改成付费测试/限定样片，并明确测试交付和正式项目抵扣规则。

这对一人公司很实用，因为我们最容易在售前阶段把小样片做成完整交付。
```

### Note 3: Delayed Materials

```markdown
# 客户一直拖资料，我用这个 Skill 生成了推进边界

客户认可方向，但多次说资料过两天给。Skill 帮我判断：这不是坏客户，但项目优先级不足。下一步不是继续设计，而是发资料清单、截止时间、启动款和重新排期规则。

这个输出让我从“继续等客户”变成“设置项目门槛”。
```

## Outreach Script

Use this when asking friends or peers to install and try it:

```text
我做了一个参赛 Skill，叫「一人公司售前作战室」。

它不是写销售鸡汤，而是把潜在客户线索和客户聊天记录转成：破冰话术、跟进节奏、报价边界、成交台账和可复用经验。

如果你有获客、接单、报价、客户拖延、免费 Demo、分成合作这类经历，可以拿一段匿名线索或聊天试一下。跑完后如果有用，帮我发一条实战笔记就行：写你输入了什么、它帮你解决了什么。
```

## 24-Hour Push Plan

1. Submit the skill with the revenue-share demo screenshot.
2. Post one flagship practical note from the HUIYANNE-style case.
3. Ask 10-20 AI freelancer / solo-founder peers to run one real client chat.
4. Give them the three note templates above.
5. Collect screenshots and objections; update `follow-up-templates.md` and `case-record-template.md` if users struggle.
6. Record or rehearse the 60-90 second demo in `demo.md`.

## Quality Gate Before Sharing

Before each public push, verify:

- output starts with `结论`, `下一步动作`, `可直接发送的话术`
- output includes a concrete message the user can send
- output includes `成交台账记录`, `可复用经验`, and `下次触发条件`
- advice does not accept free complete demo
- advice does not replace current build fee with future revenue share
- note draft is anonymized

## Top-3 Risk Register

| Risk | Fix |
|---|---|
| sounds like generic sales assistant | emphasize deal memory and boundary-setting |
| judged as not acquisition-focused | lead with acquisition + follow-up and show first-message demo |
| output too long | keep first three sections short and sendable |
| users do not post notes | give ready-to-edit practical-note templates |
| weak demo | use the revenue-share case, not a feature list |
| judged as personal-only | show three service lanes: AI workflow, brand website, content acquisition |
