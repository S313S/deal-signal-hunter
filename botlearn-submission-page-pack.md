## Long Description

一人公司经常不是不会交付，而是卡在售前：不知道找哪类客户、第一句话怎么破冰；客户问价格时不知道怎么拆报价；客户提出免费 Demo、后续分成、拖延资料时不知道怎么设边界；每次谈单经验都散落在聊天记录里，下次又重新判断。

「成交信号捕手」把潜在客户线索和客户聊天记录转成一份可执行的售前作战单。

获客场景下，它会输出：

1. 目标客户画像
2. 最值得切入的痛点
3. 首封破冰话术
4. 三步跟进节奏
5. 线索评分表
6. 可复用获客经验

跟进/谈单场景下，它会输出：

1. 结论
2. 下一步动作
3. 可直接发送的话术
4. 报价/边界建议
5. 成交台账记录
6. 可复用经验
7. 下次触发条件

它不是普通销售话术生成器，而是面向 AI 自由职业者、一人公司、小型工作室的售前判断框架。核心目标是：让每一次客户沟通不只是“回一条消息”，而是沉淀成后续还能复用的销售经验资产。

## What Problem It Solves

```text
没有销售团队，获客、跟进、报价、边界和成交复盘都靠自己。
```

具体解决：

- 潜在客户很多，但不知道谁最值得联系
- 第一封外联消息容易写得太泛
- 客户问价格时容易直接报死价
- 客户要求免费完整 Demo 时容易过度交付
- 客户拖延资料和付款时缺少推进边界
- 客户提出未来分成时，容易替代当前基础建设费
- 每次谈单经验没有沉淀，下次又重新踩坑

## Why It Fits OPC

OPC 最大的问题不是单点能力，而是没有销售、BD、项目经理和 CRM 团队。这个 Skill 把这些售前动作压缩成一个可复用流程：

```text
线索判断 → 破冰话术 → 跟进节奏 → 报价边界 → 成交台账 → 经验沉淀
```

## Example Prompt 1: Revenue Share Boundary

```text
客户项目类型：
品牌独立站 + AI 视觉 + 前端上线

客户最新表达：
费用能不能再优化一些？我们现在也是初期阶段。后面如果网页产生营业额，你可以占一定百分比，这样对双方长期合作也比较好。

我已经做了什么：
已经帮客户做了品牌视觉方向、AI 氛围图、页面结构、文案和前端预览。之前只有少量辛苦费，还没有正式结算。

我最纠结的问题：
我不想直接拒绝长期合作，但也不想让未来不确定分成替代当前建设费。
```
