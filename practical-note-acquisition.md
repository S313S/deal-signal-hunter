# 我用「成交信号捕手」生成了一条获客破冰消息

## 背景

一人公司做获客时，最容易犯的错误不是“不发消息”，而是发出去的话太泛：

- 我可以帮你做 AI
- 我可以帮你做网站
- 我可以帮你做内容自动化

这些话客户很难马上理解为什么和自己有关。

所以我用「成交信号捕手」试了一个获客场景：给一个小型品牌线索生成首封破冰和三步跟进节奏。

## 我输入给 Skill 的内容

```text
潜在客户类型：
小型品牌方，有官网/社媒内容，但页面展示和客户跟进路径比较弱。

我能提供：
AI 工作流、品牌独立站、内容自动化和售前跟进经验沉淀。

我最纠结的问题：
我不想一上来就说“我能帮你做 AI”，想找到一个客户更容易理解的切入点。
```

## Skill 输出的方向

| 模块 | 输出 |
|---|---|
| 目标客户画像 | 有活动/产品/内容，但展示、复用和跟进链路弱的小型品牌 |
| 最值得切入的痛点 | 不是“多做内容”，而是把展示、获客和跟进变成稳定流程 |
| 首封消息 | 先指出一个具体观察，再提出一个很小的示例输出 |
| 跟进节奏 | Day 0 破冰，Day 2-3 补案例，Day 6-7 清爽收口 |
| 可复用经验 | 不要卖 AI，卖客户看得懂的流程改善 |

## 可直接发送的破冰话术

```text
你好，我看了你们最近的官网和内容，感觉你们现在最值得优化的不是单纯多发内容，而是把品牌展示、客户咨询和后续跟进做成更稳定的转化流程。

我这边做的是 AI 工作流和一人公司售前自动化，可以先基于你们公开信息做一个很小的示例：比如一页品牌展示/跟进流程建议。

如果你愿意，我可以先发你一页简版，不占用太多时间。
```

## 对一人公司的价值

这次最有用的是它没有让我泛泛地“写一封开发信”，而是先帮我判断：

- 谁值得联系
- 为什么现在联系
- 第一条消息应该切哪个痛点
- 后面如果对方没回，怎么跟进而不是一直追问

这让获客动作从“随机私信”变成了一个可以复盘和沉淀的流程。

## 可复用经验

```markdown
### 获客经验：不要卖 AI，卖客户看得懂的流程改善

- 目标客户：有品牌展示或内容更新压力的小型品牌方。
- 触发信号：官网/内容长期不更新，但近期有活动或新品。
- 有效破冰点：指出展示、跟进、内容复用的具体断点。
- 无效说法：直接说我能帮你做 AI 自动化。
- 下次复用场景：客户公开内容多，但转化路径和跟进动作弱。
```
