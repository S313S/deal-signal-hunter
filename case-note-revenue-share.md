# 我用「成交信号捕手」处理了一次“后续分成”谈判

## 背景

我是一个接 AI 工作流、品牌独立站和内容自动化项目的一人公司型服务者。

最近遇到一个很典型的情况：客户认可前期做出来的品牌独立站方向，也愿意继续长期合作，但在谈到当前阶段费用时，希望“费用再优化一些”，同时提出后续可以按网页营业额给一定比例分成。

这个问题看起来是价格问题，其实更像是一人公司很容易踩的售前边界问题：

- 当前建设成本已经发生了
- 后续营业额还没有产生
- 分成口径、结算周期、归因方式都没有定义
- 如果直接答应，可能会把已经完成的制作成本变成未来不确定收益

## 我输入给 Skill 的内容

```text
客户项目类型：
品牌独立站 + AI 视觉 + 前端上线

客户最新表达：
费用能不能再优化一些？我们现在也是初期阶段。
后面如果网页产生营业额，你可以占一定百分比，这样对双方长期合作也比较好。

我已经做了什么：
已经帮客户做了品牌视觉方向、AI 氛围图、页面结构、文案和前端预览。
之前只有少量辛苦费，还没有正式结算。

我最纠结的问题：
我不想直接拒绝长期合作，但也不想让未来不确定分成替代当前建设费。
```

## Skill 先给的结论

它没有一上来写一大段情绪化回复，而是先把问题拆清楚：

| 判断 | 内容 |
|---|---|
| 客户不是直接拒绝 | 客户仍然认可长期合作 |
| 核心风险 | 用未来不确定收益替代当前建设费 |
| 当前动作 | 把基础建设费和后续分成拆开 |
| 让步方式 | 如果降价，必须同步缩小范围或限定维护期 |

这个结论对我有用，因为它把我的关注点从“怎么解释自己做得很辛苦”，拉回到“怎么设置商业边界”。

## Skill 给出的可发送话术

```text
姐姐，分成这个方向我觉得后面可以聊，也感谢你们是往长期合作考虑。

不过我建议不要用后续营业额分成来替代当前网站的基础建设费用。因为现在还没有正式上线，也还没有稳定营业额，前期这部分主要是已经发生的品牌页面、视觉素材、文案结构、前端搭建和上线准备成本。

比较稳的方式是：当前阶段先按基础建设费结算；后续如果正式上线并产生可核算订单，我们再单独谈分成比例、统计口径、结算周期，以及我具体负责哪些增长动作。
```

这段话术的好处是：没有把客户推开，也没有直接接受一个对服务方不稳定的合作结构。

## 它顺手沉淀了成交台账

```markdown
### 最新进展

客户希望优化当前费用，并提出后续按网页营业额分成。

### 客户心理

客户认可长期合作方向，但当前现金付款优先级不足，倾向用未来收益降低前期固定支出。

### 推荐报价策略

基础建设费和后续分成分开。当前阶段先结算已完成的品牌页面、视觉素材、文案结构、前端搭建和上线准备；分成只作为网站上线、有可核算订单后的增值合作。
```

这部分是我觉得最适合一人公司的地方。它不是只解决当前一句话怎么回，而是帮我把客户状态沉淀下来，方便后续继续追踪。

## 它还沉淀了一条可复用经验

```markdown
### 经验：分成不能替代基础建设费

- 触发条件：客户提出未来营业额分成，同时希望当前费用优化。
- 错误做法：口头答应分成并继续免费推进。
- 推荐做法：当前建设费先结算，后续分成另谈统计口径、周期和责任范围。
- 可复用话术：分成后面可以聊，但建议和当前基础建设费用分开。
```

下次只要遇到类似表达：

- “后面有订单了给你分”
- “先低价做，长期合作空间很大”
- “现在初期没预算，后续按收益算”

我就知道不能只看成客户在压价，而要先判断：**是不是在用未来不确定收益替代当前已经发生的基础工作。**

## 对一人公司的价值

这次最有用的不是一段话术，而是三个动作：

1. 先给结论，减少我在客户消息里反复纠结。
2. 给出可以直接发送的边界话术。
3. 把这次商谈沉淀成下次能复用的经验。

一人公司最容易遇到的问题是：客户沟通都在微信里，经验都散在脑子里。每次遇到新客户，又重新判断、重新焦虑、重新解释。

这个 Skill 的价值是把每次客户跟进变成一个小型经验资产。

## 我会怎么继续优化

后续我希望继续把更多真实谈单情况沉淀进去，比如：

- 客户要求免费完整 Demo
- 客户迟迟不补资料
- 客户问为什么这么贵
- 客户想把维护、源码、素材都默认包含

这些不是单纯销售话术问题，而是一人公司能不能稳定成交、稳定交付、稳定保护边界的问题。

## 推荐使用方式

你可以直接把下面这个结构丢给 Skill：

```text
客户项目类型：
客户最新聊天记录：
我已经做了什么：
我最纠结的问题：
```

然后重点看它有没有输出：

- `结论`
- `下一步动作`
- `可直接发送的话术`
- `成交台账记录`
- `可复用经验`
- `下次触发条件`

如果这几项都有，它就不是只在帮你“写回复”，而是在帮你搭一套自己的谈单经验库。
