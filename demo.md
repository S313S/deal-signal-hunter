# Deal Signal Hunter Demo

## Demo 输入

```text
客户项目类型：品牌独立站 + AI 视觉 + 前端上线

客户最新表达：
费用能不能再优化一些？我们现在也是初期阶段。后面如果网页产生营业额，你可以占一定百分比，这样对双方长期合作也比较好。

我已经做了什么：
我已经帮客户做了品牌视觉方向、AI 氛围图、页面结构、英文文案和前端预览。之前只有少量辛苦费，还没有正式结算。

我最纠结的问题：
我不想直接拒绝长期合作，但也不想让未来不确定分成替代当前建设费。
```

## 理想输出示例

### 结论

这不是直接拒绝信号，而是客户想降低前期现金支出，用未来不确定收益替代当前已经发生的建设成本。

可以谈长期合作，但不要让分成替代当前基础建设费。

### 下一步动作

把合作拆成两层：

| 层级 | 处理方式 |
|---|---|
| 当前阶段 | 基础建设费先结算，覆盖已完成的品牌、视觉、页面和上线准备 |
| 后续合作 | 网站上线、有可核算订单后，再谈分成或维护合作 |

如果要让步，只在价格和范围之间做取舍，不扩大免费交付。

### 可直接发送的话术

```text
姐姐，分成这个方向我觉得后面可以聊，也感谢你们是往长期合作考虑。

不过我建议不要用后续营业额分成来替代当前网站的基础建设费用。因为现在还没有正式上线，也还没有稳定营业额，前期这部分主要是已经发生的品牌页面、视觉素材、文案结构、前端搭建和上线准备成本。

比较稳的方式是：当前阶段先按基础建设费结算；后续如果正式上线并产生可核算订单，我们再单独谈分成比例、统计口径、结算周期，以及我具体负责哪些增长动作。
```

### 报价/边界建议

| 项目 | 建议 |
|---|---|
| 当前基础建设费 | 不建议用分成替代，可给首次合作友好价 |
| 上线部署 | 可包含基础上线，也可单独列项 |
| 维护 | 最多明确 1-2 个月轻维护，期满另算 |
| 不包含 | 电商支付、长期运营、批量上新、SEO、无限修改、素材授权 |

```markdown
### 最新进展

客户希望优化当前费用，并提出后续按网页营业额分成。

### 客户心理

客户认可长期合作方向，但当前现金付款优先级不足，倾向用未来收益降低前期固定支出。

### 推荐报价策略

基础建设费和后续分成分开。当前阶段先结算已完成的品牌页面、视觉素材、文案结构、前端搭建和上线准备；分成只作为网站上线、有可核算订单后的增值合作。
```

### 可复用经验

```markdown
### 经验：分成不能替代基础建设费

- 触发条件：客户提出未来营业额分成，同时希望当前费用优化。
- 错误做法：口头答应分成并继续免费推进。
- 推荐做法：当前建设费先结算，后续分成另谈统计口径、周期和责任范围。
- 可复用话术：分成后面可以聊，但建议和当前基础建设费用分开。
```

### 下次触发条件

当客户同时出现这两个信号时复用这条经验：

- 认可长期合作，但当前固定费用想压低
- 用未来订单、营业额、资源置换或长期机会替代当前建设费
